stringwidth

string stringwidth wx wy

calculates the change in the current point that would occur if string were given as the operand to show with the current font. wx and wy are computed by adding together the width vectors of all the individual glyphs for string and converting the result to user space. They form a distance vector in the x and y dimensions describing the width of the glyphs for the entire string in user space. See Section 5.4, "Glyph Metric Information," for a discussion of glyph widths.

To obtain the glyph widths, stringwidth may execute the descriptions of one or more of the glyphs in the current font and may cause the results to be placed in the font cache. However, stringwidth prevents the graphics operators that are executed from painting anything onto the current page.

Note that the width returned by stringwidth is defined as movement of the current point. It has nothing to do with the dimensions of the character outlines (see charpath and pathbbox).

Errors: invalidaccess, invalidfont, rangecheck, stackunderflow, typecheck

See Also: show, setfont
