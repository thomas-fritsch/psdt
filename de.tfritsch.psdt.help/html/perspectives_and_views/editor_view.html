Editor View

The Postscript Editor View is part of the PostScript Perspective and of the Debug perspective. It displays the PostScript file.

Notice the syntax highlighting. Different kinds of elements in the PostScript are rendered in unique colors:

  • Regular comments
  • DSC comments
  • Numbers
  • Strings
  • Literal names
  • Executable names

Notice also the code folding you get for multi-line procedures {...}, arrays [...] and dictionaries <<...>>.

Content assist

Type Ctrl-Space to get content assist for the current curor position.

Documentation Hovers

There are 2 alternative ways to get a context-sensitive documentation tooltip.

  • Put the text cursor into a name in the editor, click F2.
  • Hover the mouse over a name in the editor, wait a second.

Documentation Hyperlinks

There are 2 alternative ways to get context-sensitive documentation showing up in a browser view.

  • Put the text cursor into a name in the editor, click F3.
  • Move the mouse over a name in the editor while holding the Control key. The name will become a clickable hyperlink. Click it

Breakpoints

Breakpoints are displayed in the left ruler. Enabled breakpoints are marked by blue circles [ ], and disabled breakpoints by white circles [ ].

Instruction Pointer

If execution is suspended, the current line is marked by a blue arrow [ ] in the left ruler, and the current token is highlighted.
